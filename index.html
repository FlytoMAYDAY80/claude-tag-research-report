Claude Tag Research Report

Claude Tag
不只是新的 @Claude

它把 Slack 从“调用 AI 的聊天入口”，变成团队共同调度、共享上下文、长期运行 AI Agent 的工作控制面。

从个人助手
到频道级 AI 同事

真正的跃迁不在“能不能 @”，而在共享记忆、主动跟进、异步执行和独立 Agent Identity。

01核心结论

Claude Tag 是 Anthropic 于 2026 年 6 月 23 日发布的团队协作 Agent，首发 Slack，面向 Claude Team 与 Enterprise 用户开放 Beta，并将替代旧 Claude in Slack 体验。

Multiplayer

同一频道成员面对同一个 Claude，任务与进展对团队可见，任何人都能接续。

四个跃迁

共享记忆、主动介入、长任务异步执行、独立 Agent 身份与审计。

当前范围

Slack only，使用 Opus 4.8；官方目标是扩展到更多团队工作场所。

一句话判断：旧版是“人在需要时调用 Claude”；Claude Tag 是“把一个有权限、有记忆、会持续工作的 Claude 放进频道”。

02定位与产生背景

更准确的产品类别是：嵌入协作空间、由多人共享、能连接组织数据和工具的通用执行 Agent。

2025-10-01

Claude in Slack（初代）

双向连接：可在 Slack 的 DM、侧边栏和 thread 调用 Claude，也可把 Slack 作为 Claude 的数据源。每位用户面对的是“自己的助手”。

2025-12-08

Claude Code in Slack

thread 中 @Claude 即可拉起完整编码 Session：读取近期讨论、选择仓库、在线修复、回贴进度，并返回会话与 PR 链接。

2026-06-23

Claude Tag

加入频道/工作区记忆、主动行为、计划任务、数小时至数天的长时运行，以及组织级 Agent Identity。

2026-08-03

旧版计划切换

帮助中心显示 Claude in Slack 将切换至 Claude Tag；频道交互采用新的身份、记忆与计费模式。

为什么会产生：工作上下文沉淀在群聊里，但 Agent 长期困在“一人一聊、一问一答”。要成为团队成员，必须补齐多人协作、持续记忆、后台执行和组织权限治理。

03主要能力与核心优势

频道内自然委派

@Claude 即可拉数、建工单、整理结论、修 Bug 或开草稿 PR。

跨线程长期记忆

记住频道与工作区相关信息；管理员可以查看、修改和删除。

异步长任务

拆解任务并持续数小时或数天，在需要人类输入时回来询问。

主动与计划任务

Ambient/standing instructions 可追踪沉默线程、监控频道和定时运行。

跨系统执行

连接 API、数据、代码库、Skills 和插件，查询、写入并返回产物。

Agent Identity

频道任务不用借用某位员工账户，而是使用管理员配置的独立服务身份。

核心优势：把模型、频道上下文、组织记忆、工具连接、长任务、权限与审计组合为一套协作运行时。

04与之前 Slack @Claude 的差别

维度原 Claude in Slack / Claude CodeClaude Tag
定位个人 Claude 的 Slack 入口；编码请求可路由到 Claude Code。驻留频道的组织级共享 Agent。
上下文频道最近 20 条、thread 最近 50 条；主要围绕单次会话。跨 thread、跨天保留频道/工作区记忆。
多人接力回答可见，但任务主要由发起人会话驱动。频道所有人面对同一个 Claude，可共同 steering 和接续。
主动性主要由用户发起；编码任务可后台运行。通用任务可长期异步运行、定时执行和主动 @ 人。
权限使用发起用户的账号、连接器和权限。频道使用组织配置的 Agent Identity；DM 仍为个人身份。
治理依赖个人限额与各系统日志。组织/频道额度、阈值预警、任务与网络调用审计。

05Claude 三代界面演进

入口一直是 @Claude，但 @ 之后启动的工作系统发生了三次变化。

2025.10 · Claude in Slack：自己的助手

2025.12 · Claude Code in Slack：编码任务路由

2026.06 · Claude Tag：频道级共享 Agent

三代演进：一次回答 → 后台编码 Session → 频道共同拥有的通用 Agent。

06同类产品：界面与机制放在一起看

各家都在把 Agent 放进协作平台，但在响应方式、可用数据、执行工具和权限主体上差别很大。

ChatGPT Agents in Slack

OpenAI · 把企业自建 workspace agents 投放进 Slack

最接近
协作平台与响应

进入 Slack 公共/私有频道及 Enterprise Grid；可仅被 @ 时回复、读取相关消息自行决定回复、继续跟进 thread，或按 schedule 主动发消息。

数据与工具

使用 Agent Builder 配置的 Tools、Skills、Files、Memory 和私有系统；官方示例包括 Gmail、Notion，可发邮件、更新文档并向 Slack 输出文件。

管理员治理

Slack 批准应用；ChatGPT Enterprise 开启 Workspace agents in Slack，并用 RBAC 控制访问。Enterprise Grid 需要组织、workspace、成员三层配置。

与 Claude Tag 差别

更像“创建多个专用 Agent 再逐个投放”。公开资料未明确频道共享长期记忆、数天长任务和独立 service identity 审计。

官方：ChatGPT Agents App in Slack

Agentforce in Slack

Salesforce · CRM 与员工服务驱动的业务 Agent

业务系统最深
协作平台与响应

在 Salesforce Builder 创建后部署到 Slack Agents 标签页、DM 和频道；通过 @mention 或 Slack Workflow 响应。

数据与工具

结合 Slack 消息/文件、Salesforce Customer 360、Data Cloud；Enterprise Search 可接 Drive、OneDrive、Asana。Slack Actions 可搜索、建 Canvas、建频道、发 DM。

管理员治理

连接 Salesforce org、映射用户，由 Slack owner/admin 审核权限并安装到 workspace；可管理 profile、成员访问和卸载，Service Agent 默认需显式授权。

与 Claude Tag 差别

围绕 Salesforce 业务对象搭建多个专用 Agent；响应遵循用户的 Slack/Salesforce 权限，而非频道 Agent Identity。

官方：管理 Agentforce · Agentforce for Employees

Glean in Slack

Glean · 企业知识检索与权限感知 Agent

知识上下文最强
协作平台与响应

进入 Slack 频道、DM 侧边栏和搜索入口；可 @Glean、检测问题自动回答、/glean 搜索、Daily Digest、主动摘要和后台 Agent 消息。

数据与工具

检索有权限的公共/私有频道、DM 和文件，并连接 100+ 企业应用。可发/定时发消息、建频道、邀请成员、设提醒；Code Writer 可生成草稿 PR。

管理员治理

在 Glean Admin Console 授权 Slack 数据源、Events 和 Gleanbot，选择自动回答范围、Actions 与成本；检索继承源系统权限。

与 Claude Tag 差别

中心是权限感知企业搜索与 system of context，未公开提供统一的频道长期身份与 Agent 记忆。

官方：Glean in Slack · Help Center

Dust in Slack

Dust · 模型中立的可配置公司 Agent 平台

定制最灵活
协作平台与响应

支持 Slack 公共/私有频道，不支持 DM；用 @Dust +agent 调指定 Agent，或绑定频道默认 Agent，也可经 Workflow 触发。

数据与工具

可接 Drive、Notion、Confluence、GitHub、Microsoft、Zendesk、Snowflake、BigQuery、Gong 和 MCP；Slack 调用只看到被提及的 thread。

管理员治理

统一管理用户角色、Spaces、Agents、Connections、Tools/MCP、个人/共享凭据、审计日志和 Credits。

与 Claude Tag 差别

企业需自行设计数据、工具、触发和记忆策略；Slack 默认缺少跨 thread 的频道长期记忆。

官方：Dust in Slack · 限制说明

Microsoft 365 Copilot in Teams

Microsoft · Teams 群聊参与者与 M365 Agent 生态

M365 原生
协作平台与响应

作为参与者进入 Teams 一对一/群聊。若来源对所有人可见则公开回答；否则发起人先收到私密预览，并 Approve 或 Reject 分享。

数据与工具

按发起人权限读取文档、SharePoint、聊天/频道历史和 Web；通过 Graph 使用邮件、会议、文件。自定义 Agent 可增加 Jira、Dynamics、Bing、连接器、API 与 Actions。

管理员治理

Teams Admin Center 控制 App 访问；Copilot Control System 可盘点、部署、阻止、移除并按用户/组分配，查看数据源和动作；Power Platform DLP 限制连接器与发布渠道。

与 Claude Tag 差别

Microsoft 的权限主体是发起用户；Claude Tag 的权限主体是频道里独立于个人的 Agent Identity。

官方：Teams 群聊 · 管理员控制

08风险与落地建议

Beta 与平台范围

当前 Slack-only，能力和管理界面仍可能变化。

新权限语义

频道成员可能通过 Agent 使用其本人没有直接权限的资源，必须按最小权限配置。

记忆污染

错误结论若写入长期记忆会持续放大，需要复核与清理机制。

主动噪音

Ambient 默认关闭；应先在低风险频道验证再逐步启用。

使用量成本

按用量计费，长任务、多频道和主动模式需设置额度。

厂商指标

“65% 产品团队代码”是 Anthropic 自述，不能直接外推为客户 ROI。

试点顺序：低风险频道 → 只读连接 → 固定三类任务 → 审计错误与成本 → 再开启写入、计划任务和 Ambient。
